Exemplo Acessível — Fluxo de Login com CPF e Senha

Objetivo

Este site demonstra um fluxo simples (Home → CPF → Senha → Sucesso) com foco em semântica, navegação por teclado e compatibilidade com leitores de tela.

Abrir o exemplo

O que torna o fluxo acessível

  • Estrutura semântica: uso de <header>, <nav>, <main>, <section>, <aside>.
  • Skip link: permite pular direto para o conteúdo principal.
  • Labels reais: <label for> associado ao <input>.
  • Erros anunciados: mensagens de erro com role="alert" e estado com aria-invalid.
  • Mensagens dinâmicas: região com role="status" e aria-live="polite".
  • Botão “mostrar senha” correto: <button> com aria-pressed e aria-controls.
  • Sem JS “quebrador” de teclado: foco e ordem de tabulação naturais.

Como leitores de tela interpretam

  • Títulos (H): navegação por headings para entender a página rapidamente.
  • Regiões: leitores pulam entre header/nav/main/aside.
  • Formulários: ao focar um input, o leitor anuncia label, tipo, obrigatório e “inválido”.
  • Alert: role="alert" interrompe para anunciar erros imediatamente.
  • Status: role="status" anuncia mensagens sem interromper agressivamente.

Publicar no GitHub Pages

  1. Suba os 5 arquivos na raiz do repositório.
  2. No GitHub: Settings → Pages.
  3. Em “Build and deployment”: escolha Deploy from a branch.
  4. Branch: main e folder: / (root).
  5. A URL publicada vai apontar para index.html automaticamente.
